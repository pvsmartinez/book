# Shifting Value Functions & Oscillation

**Source:** Voice Transcript (Session 2)
**Related to:** [[time_and_compounding]], [[the_adaptation_equation]], [[emergent_behavior]]

## Core Concept
The "Value Function" (the rule deciding what survives) is not always static.
*   **Static Value Function:** Leads to **Extremism**. The system optimizes endlessly in one direction until it breaks.
*   **Dynamic Value Function:** Leads to **Oscillation**. The system swings back and forth as the environment reacts to the agents.

## 1. The Trap of the Static Function (Extremism)
If the rules never change, the "Winner" becomes a caricature of optimization.
*   **The Cheetah Paradox:**
    *   *The Goal:* Be faster to catch prey.
    *   *The Optimization:* Cheetahs evolved to be incredibly fast.
    *   *The Cost:* They became hyper-specialized (lightweight, overheat easily).
    *   *The Consequence:* They are now so specialized that they cannot defend their kill. Hyenas (who optimized for group strength and endurance) steal their food while the Cheetah is cooling down.
    *   *Lesson:* **Over-optimization creates fragility.** The system pushed "Speed" so far that it broke the "Survival" metric.

## 2. The Feedback Loop (Oscillation)
Often, the success of an agent changes the environment, which then changes the value function.
*   **Generational Cycles (The Sine Wave):**
    *   *Parents:* "We were repressed, so we will be open."
    *   *Children:* "Our parents were too loose/messy, so we will be conservative/structured."
    *   *Result:* Culture oscillates between repression and freedom, minimalism and maximalism.
*   **Fashion:** What is "desirable" is often just "what is rare."
    *   If everyone wears Blue, Red becomes valuable.
    *   If everyone wears Red, Blue becomes valuable.
    *   The value function is inverted based on the current state of the population.

## 3. The System Designer's Role
When we see a system compounding towards an unhealthy extreme (e.g., Wealth Inequality, Hyper-Polarization), it means the **Value Function is Static**.
*   **The Fix:** We cannot just ask the agents to "be better" (Cheetahs can't just decide to be strong). We must **edit the environment** to introduce a counter-force or oscillation.
*   **Example:** If a game meta becomes "Only Snipers," the developer (System Designer) changes the map to have more cover. The Value Function shifts from "Aim" to "Positioning."

## Key Takeaway
**Time doesn't just compound quantity; it compounds the pressure on the Value Function.** Eventually, the function must break or shift.
